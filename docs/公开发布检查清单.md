# 公开发布检查清单

本文用于发布 `spectrace-rd` npm 包前自查。

## 必须检查

- `package.json` 已包含 `name`、`version`、`description`、`license`、`author`、`repository`、`homepage`、`bugs`、`engines`、`publishConfig`。
- `README.md` 首屏说明清楚：这是什么、适合什么场景、如何 5 分钟开始。
- `LICENSE`、`CHANGELOG.md`、`CONTRIBUTING.md`、`SECURITY.md` 已存在。
- `npm run check` 通过。
- `npm run smoke` 通过。
- `npm run pack:check` 通过，并确认包内不包含本地工作区产物、缓存文件、密钥、个人绝对路径。
- `node bin/spectrace.js --help` 正常。
- `node bin/spectrace.js version` 正常。
- `node bin/spectrace.js init --yes --install none --workspace <tmp> ...` 正常。
- `node bin/spectrace.js doctor --workspace <tmp>` 正常。

## CLI 能力

公开包至少应提供：

- `spectrace init`：交互式初始化。
- `spectrace doctor`：检查本机环境和工作区配置。
- `spectrace install`：安装 skill 到 Codex / Claude。
- `spectrace sync`：强制同步当前包内 skill 到 Codex / Claude。
- `spectrace migrate`：补齐旧工作区缺失配置项。
- `spectrace version`：显示版本。

## 技术规范建议

- 保持 CLI 无第三方运行时依赖，降低安装失败概率。
- 所有本地路径默认支持相对路径和绝对路径。
- 不把通知 webhook、token、cookie 写入仓库，只放在用户本机配置。
- 不让 AI 手写状态文件、验证报告、通知记录等标准产物。
- npm 包只发布 skill、脚本、文档和许可证，不发布工作区中间产物。
- 发布前用 CI 在 macOS、Windows、Linux 上跑 smoke test。

## 仍可继续增强

- 增加 `spectrace templates`，列出 Java / Node / Go / Python / 通用存量系统模板。
- 增加 `spectrace doctor --fix`，自动创建缺失目录或提示最小修复命令。
- 增加 E2E fixture，完整验证 `openspec + auto` 和 `todo + auto` 状态流。
- 增加 npm provenance / signed publishing，提升供应链可信度。
