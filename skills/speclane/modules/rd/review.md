# 代码审查清单

review 是门禁，不只是报告。每条发现应尽量给出严重级别、范围、原因和建议修复方向。

优先检查这些问题：

- 需求覆盖：实际改动是否覆盖 todo 里的每个未完成任务
- 计划一致性：计划与代码差异之间是否存在不一致
- 范围控制：本次改动是否意外扩大了范围
- 行为正确性：核心逻辑、边界场景、非法输入路径是否合理
- 数据兼容：接口、DB、配置、序列化、消息字段是否兼容
- 测试充分性：应用代码改动后，测试或人工验证是否同步更新
- 规范遵守：参考文件里的业务规则和开发规范是否仍被遵守

阻塞级问题：

- 没有检测到任何代码差异
- 明确缺失 todo 中的核心需求
- 改动会破坏已知兼容性
- 验证前必须修复的编译、测试或配置问题

非阻塞问题：

- 计划范围过宽
- 部分计划内文件最终无需修改
- 缺少测试但已有明确人工验证说明
