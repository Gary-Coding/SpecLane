# 计划阶段规则

计划阶段由 `discover -> plan` 两步组成。

## discover

`discover` 负责把 todo 中的自然语言需求转成代码定位证据。

必须产出：

- `discovery.json`
- `discovery.md`

定位优先级：

1. todo 中明确写出的服务名、仓库名
2. 接口路径、方法名、类名、字段名、表名、MQ topic、配置 key
3. 业务关键词
4. 项目结构和构建文件

如果关键词没有命中代码，不应伪造影响文件。应在 `planning_hints` 中说明计划阶段需要继续定位入口。

## plan

`plan` 必须基于 `discovery.json`、todo 和参考文件生成。

计划至少包含：

- 需求摘要
- 任务拆解
- 上下文定位证据
- 计划置信度：`high`、`medium`、`low`
- 影响仓库、模块、文件
- 每个任务的验收标准
- 可独立推进的实施切片
- 测试计划
- 风险和未知项

置信度建议：

- `high`：关键词多处命中代码，且影响文件明确
- `medium`：有部分命中，或能通过项目结构推断影响范围
- `low`：缺少有效命中，只能先按结构继续定位

计划置信度低时，`auto` 模式也不应停下来请求确认；应继续在实现阶段定位，但要把低置信度和未知项写入计划。
